# DevOps 契约测试

基于 PRD 用户故事的 QA 测试文档。

## 测试 1：定义约束

### TC 1.1：添加新约束

```
场景：在 Contract 中添加新约束
输入：禁止 force push 到 main
期望：
- [ ] 约束成功添加
- [ ] 可通过命令查询到约束
- [ ] 约束持久化到存储
```

### TC 1.2：删除约束

```
场景：删除已有的约束
输入：约束 ID
期望：
- [ ] 约束成功删除
- [ ] 查询结果为空
```

### TC 1.3：修改约束

```
场景：修改约束内容
输入：约束 ID，新内容
期望：
- [ ] 约束成功修改
- [ ] 修改后的约束可查询
```

期望：

## 测试 3：用例测试

### TC 3.1：约束触发测试

```
场景：测试"禁止 force push"约束
输入：模拟 git push --force 命令
期望：
- [ ] 测试检测到违规
- [ ] 测试返回失败
- [ ] 有清晰的错误信息
```

### TC 3.2：边界触发测试

```
场景：测试"何时用 feature 分支"边界
输入：应使用 feature 分支的场景
期望：
- [ ] 边界返回通过
- [ ] 提示应使用 feature 分支
```

### TC 3.3：边界拒绝测试

```
场景：测试边界拒绝
输入：不应使用 feature 分支的场景（如 hotfix）
期望：
- [ ] 边界返回拒绝
- [ ] 提示应使用 hotfix 分支
```

## 测试 4：检查约束

### TC 4.1：push 前检查

```
场景：在 git push 前运行约束检查
输入：git push 命令
期望：
- [ ] 检查运行并返回结果
- [ ] 如违规，阻止操作
- [ ] 提供错误提示
```

### TC 4.2：多约束检查

```
场景：同时检查多个约束
输入：复杂操作场景
期望：
- [ ] 所有相关约束都被检查
- [ ] 返回所有违规项
- [ ] 全部通过才允许操作
```

### TC 4.3：skip 检查

```
场景：紧急情况需要跳过检查
输入：--skip-check 参数
期望：
- [ ] 可选跳过检查
- [ ] 跳过记录到日志
- [ ] 需要确认
```

### TC 5.3：分享文档

期望：
