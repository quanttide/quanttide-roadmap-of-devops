# DevOps 契约系统

作为开发者，我需要自定义工作规范，以便团队一致执行。

## 用户故事

### story 1：定义约束

作为开发者，我需要在Contract中声明约束，以便团队遵守。

```
场景：我想声明"禁止 force push 到 main"
验收：
- [ ] 可在 Contract 中添加新约束
- [ ] 约束格式为：禁止 <行为>
- [ ] 约束可被查询
```

### story 2：声明边界

作为开发者，我需要声明边界，以便明确什么能做、什么不能做。

```
场景：我想声明"何时用 feature 分支"
验收：
- [ ] 可声明边界条件和场景
- [ ] 边界格式为：何时 <条件>
```

### story 3：用例测试

作为开发者，我需要对约束进行测试，以验证约束有效性。

```
场景：我想测试"禁止 force push"约束
验收：
- [ ] 可创建测试场景
- [ ] 可执行测试
- [ ] 测试结果通过/失败
```

### story 4：检查约束

作为开发者，我需要在操作前检查是否违反约束。

```
场景：我想在 push 前检查是否违规
验收：
- [ ] 可在操作前运行检查
- [ ] 检查结果可阻止操作
- [ ] 有明确的错误提示
```

### story 5：生成文档

作为开发者，我需要从约束生成文档，以便团队查看。

```
场景：我想生成团队规范文档
验收：
- [ ] 可从约束生成 Markdown
- [ ] 文档包含全部约束和边界
- [ ] 文档可分享
```

---

## 验收标准

- [ ] 可添加/删除/修改约束
- [ ] 约束可被程序检查
- [ ] 检查可集成到工作流
- [ ] 文档可导出
